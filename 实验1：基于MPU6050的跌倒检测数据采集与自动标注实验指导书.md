# 基于MPU6050的跌倒检测数据采集与自动标注实验指导书

# 一、实验目的

- 理解传感器误差：掌握MPU6050六轴传感器的零偏误差（Zero-bias）产生原因及静态校准方法。

- 掌握串口通信：学习使用Python的pyserial库与底层硬件（如Arduino/STM32）进行实时数据交互。

- 理解特征提取：学习合加速度（SVM, Signal Vector Magnitude）的物理意义及其在突发运动检测中的应用。

- 掌握时间序列处理：理解“滑动窗口/环形缓冲区（Ring Buffer）”机制，学习如何捕获事件发生前后的完整上下文数据，实现自动化打标签。

## 步骤 2：解析底层硬件数据 (正则表达式)

### 【原理解析】

下位机（单片机）通过串口发送的是字符串文本，我们需要将其提取为纯数字，才能进行数学运算。

### 【代码映射】

程序使用正则表达式（Regex）扫描每一行文本，精准提取出6个维度的数值（X/Y/Z的加速度和角速度）。

```python
def parse_mpu6050_line(line):
    # 正则规则：寻找 "accel.x=数字" 的格式，并支持负号和可选的逗号
    pattern = r"accel\.x\s*=\s*(-?\d+),?\s*accel\.y\s*=\s*(-?\d+)..."
    match = re.search(pattern, line)
    if match:
        return [float(x) for x in match.groups()] # 转换为浮点数数组
```

## 步骤 3：执行零偏校准 (Calibration 模式)

### 【操作指南】

1. 将佩戴了MPU6050的设备水平静置在坚固的桌面上。

2. 在终端运行命令：`python script.py --mode calib --port COM3`（请将COM3替换为你的实际串口号）。

3. 保持绝对静止，等待程序采集500个样本。程序结束后，目录下会生成`mpu_offsets.json` 文件。

### 【代码映射 - run_calibration 函数】

- 采集静止状态数据：通过 `while count < samples:` 循环读取500次数据。

- 计算平均误差（零偏）：理想情况下，静置时除Z轴加速度（受重力影响，约等于1g或响应的AD值）外，其余轴应为0。但实际传感器有偏差，我们计算静止时的平均值作为偏差值（Offset）。

```python
offsets = {
    "ax_offset": sum(accel_x) / samples,
    "ay_offset": sum(accel_y) / samples,
    # 注意：Z轴包含重力(1g)，静态下不能强行归零，因此代码中省略了 az 的校准
    "gx_offset": sum(gyro_x) / samples,
    ...
}
```

持久化保存：使用 `json.dump()` 将校准值存入文件，供后续采集使用。

## 步骤 4：跌倒数据采集与自动标注 (Collect 模式)

### 【操作指南】

1. 确保校准文件`mpu_offsets.json` 已存在。

2. 将设备固定在人体（如腰部或手腕）。

3. 运行命令：`python script.py --mode collect --port COM3 --duration 300 --file my_dataset.csv`。

4. 在5分钟（300秒）内，执行日常动作（走、坐、站）以及模拟跌倒动作。程序会自动判断并在终端提示“触发跌倒事件”。

### 【核心逻辑与代码映射 - run_collection 函数】

#### 4.1 加载校准与数据补偿

每次读取到新数据 raw_data 时，程序会减去之前测量的零偏，得到真实数据 calib_data。

```python
calib_data = [
    raw_data[0] - offsets['ax_offset'],
    # ... az 保留原始值 ...
]
```

#### 4.2 计算 SVM (合加速度)

原理：无论朝哪个方向跌倒，三轴加速度的矢量和会产生剧烈波动。SVM 将三维矢量简化为一维标量，便于设定阈值。

```python
# 对应代码中的 def calculate_svm(ax, ay, az): 
svm = calculate_svm(ax, ay, az) # √(ax² + ay² + az²)
```

#### 4.3 核心状态机：基于环形缓冲区的自动标注

为了让机器学习模型能预测跌倒，我们不仅需要跌倒触地那一瞬间的数据，还需要**跌倒前兆（失重过程）和跌倒后（倒地不起）**的数据。程序使用双端队列（Deque）巧妙地实现了这一点。

##### 状态 A：日常活动状态 (Normal)

当 svm 未超过阈值时（未跌倒）：

- 标签标记为 0。

- 将当前数据存入 buffer（容量为50帧，约1秒的历史记录）。这就像行车记录仪，永远保留最近1秒的数据。

- 为了防止日常数据过多导致样本不平衡，代码进行了下采样（每5帧记录1帧：normal_data_skip % 5 == 0）。

```python
buffer.append(calib_data) # 记录跌倒前兆
```

##### 状态 B：跌倒触发瞬间 (Impact)

当 svm > IMPACT_THRESHOLD (撞击阈值) 时：

- 状态切换：is_falling = True。

- 关键动作：将 buffer 中积攒的过去1秒数据，全部打上标签 1（跌倒态）并写入 CSV。这保证了我们捕获了跌倒的起始过程！

- 设置跌倒后置计数器 post_fall_counter = POST_FALL_FRAMES。

```python
while len(buffer) > 0:
    writer.writerow(buffer.popleft() + [1]) # 将过去的记录回溯打上跌倒标签
```

##### 状态 C：跌倒后延续记录 (Post-fall)

跌倒发生后的接下来 50 帧（由 post_fall_counter 控制）：
即使 svm 已经恢复平稳（人躺在地上），依然强制打上标签 1 并写入文件。

```python
writer.writerow(calib_data + [1]) 
post_fall_counter -= 1
```

当计数器归零，代表一次完整跌倒事件记录完毕，清空 buffer，恢复状态 A。

# 四、课后思考与调整建议

1. 阈值调整：代码中的 `IMPACT_THRESHOLD = 2500` 是硬编码的。如果你的传感器量程（如 ±2g 或 ±8g）不同，这个值需要根据终端打印的 SVM 实时数值进行调整。

2. 如何改进判断逻辑？ 单凭瞬间的 SVM 大于阈值可能导致误判（例如用力拍桌子、跳跃）。如何利用角速度（Gyro）数据来进一步确认是“跌倒”而不是“跳跃”？

3. Z 轴为什么不归零？ 在地球上，设备静止时Z轴（假设Z轴朝上）会受到 -9.8 m/s² 的重力加速度。如果强行减去这个值，会导致算法无法感知设备的真实姿态（倾角）。
